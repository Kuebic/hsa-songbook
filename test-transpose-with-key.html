Test: Transpose with {key:} Directive

Test Case 1: Song with {key: G}

Original ChordPro: 
{title: Test Song}
{key: G}

[Verse]
[G]This is a [C]test [D]song in [G]G major
Transposition: 0
Expected Behavior:
  • At transpose = 0: Display shows "Key: G", chords are [G] [C] [D] [G]
  • At transpose = +1: Display shows "Key: G#/Ab", chords are [G#] [C#] [D#] [G#]
  • At transpose = -1: Display shows "Key: F#/Gb", chords are [F#] [B] [C#] [F#]

Test Case 2: Song with {key: C}

Original ChordPro: 
{title: Another Test Song}
{key: C}

[Chorus]
[C]Starting in [F]C major, [G]moving to [C]tonic
Transposition: 0
Expected Behavior:
  • At transpose = 0: Display shows "Key: C", chords are [C] [F] [G] [C]
  • At transpose = +2: Display shows "Key: D", chords are [D] [G] [A] [D]
  • At transpose = -2: Display shows "Key: Bb", chords are [Bb] [Eb] [F] [Bb]

Summary

Current Implementation Status:

  • ✅ ChordPro content with {key:} directive is parsed correctly
  • ✅ Original key is extracted from the content
  • ✅ Transposition hook uses the original key as base
  • ✅ When transpose = 0, chords remain unchanged
  • ✅ When transpose = ±n, all chords shift by n semitones
  • ✅ The display shows the transposed key correctly

How it works:

  1. Song has {key: G} directive
  2. useTransposition hook receives "G" as originalKey
  3. At transpose = 0, currentKey = "G" (no change)
  4. At transpose = +1, currentKey = "G#" (up 1 semitone)
  5. ChordSheetJS applies the transposition to all chords
  6. Display shows the currentKey value
